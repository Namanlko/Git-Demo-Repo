Getting Started with Git

Git is a distributed version control system that helps developers track changes in their code and collaborate with others.

$ git --version
git version 2.32.0

$ git config --global user.name "Your Name"
$ git config --global user.email "your.email@example.com"

Initialize a Repository

git init

This creates a new Git repository in your current directory.

Basic Git Commands

Tracking Changes

git add filename

Stage a specific file for commit.

git add .

Stage all changed files.

Committing Changes

git commit -m "Commit message"

Commit your staged changes with a message.

git status

View the status of your working directory.

Viewing History

git log

Show commit history.

git log --oneline --graph

Compact view with branch visualization.

Branching & Merging

Branches allow you to develop features, fix bugs, or safely experiment with new ideas in a contained area of your repository.

Create & Switch Branches

git branch feature-x

Create a new branch named "feature-x".

git checkout feature-x

Switch to the "feature-x" branch.

git checkout -b hotfix

Create and switch to a new "hotfix" branch.

Merging

git merge feature-x

Merge "feature-x" branch into current branch.

git branch -d feature-x

Delete the "feature-x" branch.

Resolving Conflicts

When merge conflicts occur:

  1. Edit the conflicted files
  2. Mark as resolved with git add
  3. Complete the merge with git commit

Working with GitHub

GitHub is a platform for hosting Git repositories and collaborating with others.

Remote Repositories

git remote add origin https://github.com/user/repo.git

Add a remote repository.

git push -u origin main

Push your local commits to GitHub (first time).

git clone https://github.com/user/repo.git

Clone an existing repository from GitHub.

Collaboration Workflows

Fork & Pull

  1. Fork the repository on GitHub
  2. Clone your fork locally
  3. Create a feature branch
  4. Push changes to your fork
  5. Open a pull request

Shared Repository

  1. Team members are added as collaborators
  2. Everyone clones the main repository
  3. Work on feature branches
  4. Push to main repository
  5. Create pull requests for review

Advanced Topics

Rebasing

git rebase main

Reapply your branch's commits on top of main branch.

Stashing

git stash

Temporarily save uncommitted changes.

git stash pop

Restore the most recently stashed changes.

Undoing Changes

git reset --hard HEAD

Discard all uncommitted changes.

git revert commit-hash

Create a new commit that undoes a previous commit.
